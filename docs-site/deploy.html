三公線上遊戲本機部署指南，提供直接執行和 Docker Compose 兩種方式。

在 GitHub 上查看完整文件 →

前置需求

工具版本說明
Node.js20.x伺服器執行環境
npm10.x套件管理
Docker24.x+容器化部署
docker-composev2.x本機多服務編排
 
# 確認安裝版本
node --version         # v20.x.x
npm --version          # 10.x.x
docker --version       # Docker version 24.x.x
docker compose version # Docker Compose version v2.x.x

方式一：直接執行（最快速）

# 1. 進入 server 目錄
cd server

# 2. 安裝依賴
npm install

# 3. 建置 TypeScript
npm run build

# 4. 啟動伺服器
npm start
伺服器啟動後監聽：ws://localhost:2567

開發模式（熱重載）

npm run dev

方式二：Docker Compose（推薦）

# 從專案根目錄執行
docker compose up --build

# 服務啟動後：
# Colyseus Server: ws://localhost:2567
# Nginx Proxy:     http://localhost:80

停止服務

docker compose down

# 清除 volumes（含 SQLite 稽核日誌）
docker compose down -v

連線測試

// 測試 Colyseus 連線
const Colyseus = require("colyseus.js");
const client = new Colyseus.Client("ws://localhost:2567");

client.joinOrCreate("sam_gong", { playerName: "TestPlayer" })
  .then(room => {
    console.log("Connected! Room ID:", room.id);
    console.log("Room Code:", room.state.roomCode);
  })
  .catch(err => console.error("Connection failed:", err));

執行測試

# 單元測試
cd server && npm test

# 覆蓋率報告
cd server && npm test -- --coverage

# E2E 測試（需先啟動伺服器）
npx playwright test

# 查看測試結果
npx playwright show-report

環境變數（選用）

# server/.env（本機開發）
PORT=2567
NODE_ENV=development
SQLITE_DB_PATH=./data/samgong.db
LOG_LEVEL=debug
MAX_ROOMS=100

常見問題

Port 2567 已被佔用

# 查看佔用 Port 的程序
lsof -i :2567
kill -9 <PID>

TypeScript 編譯錯誤

# 清除 build 目錄重新編譯
rm -rf server/dist
cd server && npm run build

Docker 映像建置失敗

# 清除 Docker cache 重新建置
docker compose build --no-cache
docker compose up

K8s 部署（生產環境）

# 部署至 Kubernetes
kubectl apply -f k8s/namespace.yaml
kubectl apply -f k8s/deployment.yaml
kubectl apply -f k8s/service.yaml
kubectl apply -f k8s/ingress.yaml

# 查看部署狀態
kubectl -n sam-gong get pods
kubectl -n sam-gong get services
