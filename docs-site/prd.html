產品願景

一行宣言

讓 2-6 人隨時隨地能公平地在線玩三公（Server-Authoritative，零作弊）。

目標對齊（BRD → PRD）

BRD 目標PRD 交付物成功指標
O1: 可玩的三公線上遊戲 完整牌局流程（創房→押注→比牌→結算） 每局完成率 ≥ 95%
O2: 穩定 Beta 用戶留存 斷線重連（60s）+ 流暢動畫 7D 留存率 ≥ 20%（Beta）
O3: 技術架構可擴展 模組化牌型引擎，可複用至其他棋牌 牌型引擎模組可獨立測試且文件覆蓋率 ≥ 80%

User Stories（摘要）

Epic 1: 房間管理

US-001: 創建房間

As 一位玩家，I want to 創建一個三公遊戲房間並獲得房間碼，So that 朋友可以用房間碼加入。

  • AC-001-1: 成功創建房間後收到 6 字元英數房間碼
  • AC-001-2: 創建者自動成為房主
  • AC-001-3: 房間碼排除易混淆字元（O/0/I/1）

US-002: 加入房間

As 一位玩家，I want to 用房間碼加入已創建的房間，So that 和朋友一起玩遊戲。

  • AC-002-1: 輸入有效房間碼後成功加入
  • AC-002-2: 房間已滿（6人）時顯示錯誤
  • AC-002-3: 無效房間碼顯示明確錯誤訊息

Epic 2: 遊戲流程

US-005: 押注

As 一位閒家玩家，I want to 在看到公牌後決定押注金額，So that 根據策略決定賭注。

  • AC-005-1: 押注金額必須在最小/最大限額內
  • AC-005-2: 超時（30s）自動棄牌
  • AC-005-3: 已押注玩家不可再次修改

US-008: 翻牌結算

As 所有玩家，I want to 看到所有人翻牌並顯示結算結果，So that 了解輸贏與籌碼變動。

  • AC-008-1: Server 比對所有玩家與莊家點數
  • AC-008-2: 公牌（點數=10）勝所有非公牌
  • AC-008-3: 平局時比花色（黑桃 > 紅心 > 方塊 > 梅花）
  • AC-008-4: 公牌平局按花色決定

遊戲狀態機

lobby
  ↓ start_game
banker_selection
  ↓ banker selected
betting
  ↓ all players bet / fold
dealing
  ↓ cards dealt
reveal
  ↓ cards revealed
settling
  ↓ settlement done
round_end
  ↓ next round / room close

共 7 個狀態，所有轉換均由 Server 控制，非法轉換直接拒絕。
