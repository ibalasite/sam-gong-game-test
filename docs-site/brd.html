技術建議

層次建議方案選擇理由
Client 框架Cocos Creator 4.x + TypeScript使用者指定；官方有 Colyseus SDK 整合文件
Server 框架Colyseus 0.15+ + Node.jsMIT 授權；天然 authoritative 設計；官方有 Cocos SDK
Server 語言TypeScript（ts-node / esbuild 打包）與 Client 統一語言，共享型別定義
資料庫SQLite（better-sqlite3）嵌入式、零配置、足夠支撐 Pilot 階段
測試Jest + Playwright業界標準；Colyseus 官方有 Test SDK

業務目標（North Star）

O1 — 可玩的三公線上遊戲

交付一個 2-6 人可即時對戰的三公線上遊戲，Server-Authoritative 確保零作弊。

  • 每局完成率 ≥ 95%
  • 延遲 ≤ 200ms（p95）

O2 — 穩定 Beta 用戶留存

透過斷線重連與流暢動畫提升留存率。

  • Beta：7D 留存率 ≥ 20%
  • GA：7D 留存率 ≥ 30%

O3 — 可擴展技術架構

模組化牌型引擎，可複用至其他棋牌遊戲。

  • 牌型引擎模組可獨立測試
  • 文件覆蓋率 ≥ 80%

投資 Go/No-Go 門檻

Go 條件：完成率 ≥ 95%、Beta 留存 ≥ 20%、無重大安全漏洞
No-Go 條件：完成率 < 90%、任何作弊漏洞、或監管合規問題未解決
