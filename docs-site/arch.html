1. 架構模式

Server-Authoritative + Client-Rendering

三公遊戲採用兩層架構：

  • Colyseus Server（唯一可信方）：所有牌局決策 100% 在 Server 執行。Client 無法偽造任何遊戲狀態。
  • Cocos Creator Client（純 Rendering）：Client 只負責接收 Server 廣播的狀態差分（State Diff）並渲染畫面，不持有任何遊戲邏輯。

3. 核心優勢

優勢說明
反作弊 未翻牌的牌面資料從未送達其他玩家的客戶端，無法透過 DevTools 竊取
單一狀態來源 SamGongState Schema 是所有客戶端視圖的唯一來源
可測試性 Server 端的純函數 Game Logic 可獨立單元測試，不依賴 Client

    image: node:20-alpine
    ports: ["2567:2567"]
    command: npm start

  nginx:
    image: nginx:1.25-alpine
    ports: ["80:80", "443:443"]
