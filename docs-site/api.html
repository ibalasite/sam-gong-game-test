三公遊戲 WebSocket API 文件，涵蓋 Room 生命週期、訊息類型及 Payload 格式。

1. 協議概覽

項目說明
通訊方式WebSocket via Colyseus Protocol (0.15)
狀態同步@colyseus/schema Differential State Sync（自動增量同步）
Client → Serverroom.send(type, payload)
Server → Client（廣播）room.broadcast(type, payload)
Server → Client（直接）client.send(type, payload)（單一目標）
反作弊@filter decorator：翻牌前對手牌面不包含在 State Patch 中

2. 連線設定

WebSocket URL:  ws://{server}/colyseus/
Room Name:      sam_gong
Colyseus Client: new Colyseus.Client("ws://server/colyseus/")

// 本機開發（直連）
const client = new Colyseus.Client("ws://localhost:2567");
Nginx 代理 /colyseus/ 至 localhost:2567。本機開發時直接連 ws://localhost:2567。

3. Room 生命週期 API

3.1 創建房間

// Client
const client = new Colyseus.Client("ws://localhost:2567");
const room = await client.create("sam_gong", {
  playerName: "Alice",
  minBet: 10,
  maxBet: 1000,
  maxPlayers: 6
});

// Response: room.id = "A3KZ7M" (6字元房間碼)
欄位類型說明
playerNamestring顯示名稱（必填）
minBetnumber最小押注（預設 10）
maxBetnumber最大押注（預設 1000）
maxPlayersnumber最大人數 2-6（預設 6）

3.2 加入房間

// Client
const room = await client.joinById("A3KZ7M", {
  playerName: "Bob"
});

// 錯誤情境
// 房間已滿: Error("Room is full")
// 無效房間碼: Error("Room not found")

4. Client → Server 訊息

4.1 start_game

// 僅房主可發送，至少 2 人才可開始
room.send("start_game", {});

4.2 place_bet

room.send("place_bet", { amount: 100 });
// amount: 必須在 minBet ~ maxBet 之間
// 狀態: 僅在 "betting" 階段有效
// 超時: 30秒未行動 → 自動折疊

4.3 fold

room.send("fold", {});
// 棄牌：押注轉讓給莊家
// 狀態: 僅在 "betting" 階段有效

4.4 reveal（Server 自動觸發）

// 所有玩家押注完畢後，Server 自動廣播翻牌指令
// Client 不需手動發送

5. Server → Client 事件

事件類型觸發時機Payload
game_started遊戲開始時{ phase: "banker_selection" }
betting_started進入押注階段{ timeLimit: 30 }
cards_revealed翻牌時{ results: [...] }
settlement_done結算完成{ changes: { sessionId: delta } }
round_ended每局結束{ nextBanker: sessionId }
error操作失敗{ code: string, message: string }

6. 錯誤碼

錯誤碼說明
INVALID_PHASE在錯誤遊戲階段發送訊息
INVALID_BET押注金額超出範圍
NOT_HOST非房主嘗試開始遊戲
RATE_LIMITED超過速率限制（10 ops/sec）
ALREADY_BET已押注，不可再次行動
ROOM_FULL房間已達人數上限
